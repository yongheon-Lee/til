Git 2주차 강의 정리

- 키워드
1. Issue
2. Branch
3. Merge

- 작업 흐름
1. Issue: 작업 담당자, 관리할 기능, 발생한 문제/버그 등을 알리고 정함

2. Branch: 프로젝트를 작업별로 나눔
기준이 되는 main 브랜치에서 나눠서 작업할 브랜치 생성 후 해당 브랜치로 체크아웃해서 작업하고 commit

3. Merge: 나눠서 작업했던 것을 하나로 합침
기준 main 브랜치로 체크아웃하고, 각 브랜치들 병합
작업 종료 후 필요 없는 브랜치는 삭제
※ 다른 브랜치에서 같은 파일 작업 후 병합 시 충돌 발생!
-> 파일 수정 후 <<<HEAD, =====, >>>브랜치명, commit 내용 부분 지운 후 commit

4. 원격 repo로 push: 로컬repo의 브랜치를 원격 repo 브랜치로 반영해 최종 업데이트!

추가 개념
- tracking: 로컬 repo와 원격 repo의 특정 브랜치를 연결하는 것
- push/pull: 로컬 repo와 원격 repo의 특정 브랜치의 commit을 반영하는 것
